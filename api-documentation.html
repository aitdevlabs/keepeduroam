Overview

KeepEduroam is a WebSocket relay service that connects providers (users sharing internet) with consumers (users needing internet access). This API documentation covers how clients can integrate with the service.

🌐 Base URL

https://keepeduroam.aitdevlabs.workers.dev

📡 Protocol

Primary: WebSocket (Socket.IO v4)
Fallback: HTTP long-polling
SSL/TLS: Enabled (Cloudflare)

📦 Response Format

JSON

🔒 CORS

All public endpoints support CORS

🔒 Authentication: All endpoints except /status and /.well-known/health require a valid JWT token obtained during device registration.

Public Endpoints

These endpoints are publicly accessible and do not require authentication. They provide service health and status information.

WebSocket API (Socket.IO)

Connection: Socket.IO v4 client required. Use transports: ['websocket'] for best performance.

Connection

# Python
import socketio
sio = socketio.Client()
sio.connect('https://keepeduroam.aitdevlabs.workers.dev', transports=['websocket'])
// JavaScript
import io from 'socket.io-client';
const socket = io('https://keepeduroam.aitdevlabs.workers.dev', {
    transports: ['websocket']
});

On successful registration, the server returns a JWT token. This token must be included in the Authorization: Bearer <token> header for all HTTP requests to protected endpoints.

Events

Client → Server Events

EventPayloadDescription
register_provider {id: string} Register as a provider (share internet)
register_consumer {id: string} Register as a consumer (use internet)
consumer_request {target, port, payload} Request a relay from a provider
relay_response {id, result} Provider returns response
ping {} Keep connection alive

Server → Client Events

EventPayloadDescription
connected {status, sid} Connection confirmed
registered {status, role, id, token} Registration confirmed with JWT
update_providers {sid: {id, points, ...}} Provider list update
update_consumers {sid: {id, data_used, ...}} Consumer list update
relay_request {id, target, port, payload} Relay request for provider
relay_result {id, result} Relay response for consumer
pong {timestamp} Ping response

Integration Examples

Python Client

# Complete KeepEduroam client in Python
import socketio
import socket
import requests

class KeepEduroamClient:
    def __init__(self, mode='provider', device_id='device_001'):
        self.server = 'https://keepeduroam.aitdevlabs.workers.dev'
        self.mode = mode
        self.device_id = device_id
        self.token = None
        self.sio = socketio.Client()
        self.setup_handlers()

    def setup_handlers(self):
        @self.sio.event
        def connect():
            print('✅ Connected to KeepEduroam')
            self.sio.emit(f'register_{self.mode}', {'id': self.device_id})

        @self.sio.event
        def registered(data):
            print(f'✅ Registered: {data}')
            # Save the JWT token
            self.token = data.get('token')

        @self.sio.event
        def relay_request(data):
            if self.mode == 'provider':
                result = self.forward_request(data)
                self.sio.emit('relay_response', {
                    'id': data['id'],
                    'result': result
                })

    def get_headers(self):
        """Return headers with JWT for HTTP requests"""
        return {'Authorization': f'Bearer {self.token}'} if self.token else {}

    def get_time(self):
        """Get stored time for this device"""
        response = requests.get(
            f'{self.server}/time/{self.device_id}',
            headers=self.get_headers()
        )
        return response.json()

    def connect(self):
        self.sio.connect(self.server, transports=['websocket'])
        self.sio.wait()

# Usage
client = KeepEduroamClient(mode='provider', device_id='phone_001')
client.connect()

JavaScript Client

// Complete KeepEduroam client in JavaScript
import io from 'socket.io-client';

class KeepEduroamClient {
    constructor(mode = 'provider', deviceId = 'device_001') {
        this.server = 'https://keepeduroam.aitdevlabs.workers.dev';
        this.mode = mode;
        this.deviceId = deviceId;
        this.token = null;
        this.socket = null;
    }

    setupSocket() {
        this.socket = io(this.server, { transports: ['websocket'] });

        this.socket.on('connect', () => {
            console.log('✅ Connected to KeepEduroam');
            this.socket.emit(`register_${this.mode}`, { id: this.deviceId });
        });

        this.socket.on('registered', (data) => {
            console.log('✅ Registered:', data);
            // Save the JWT token
            this.token = data.token;
        });

        this.socket.on('relay_request', async (data) => {
            if (this.mode === 'provider') {
                try {
                    const res = await fetch(`http://${data.target}:${data.port}`, {
                        method: 'POST',
                        body: data.payload
                    });
                    const result = await res.text();
                    this.socket.emit('relay_response', {
                        id: data.id,
                        result: result
                    });
                } catch (error) {
                    this.socket.emit('relay_response', {
                        id: data.id,
                        result: `Error: ${error.message}`
                    });
                }
            }
        });
    }

    getHeaders() {
        // Return headers with JWT for HTTP requests
        return this.token ? { 'Authorization': `Bearer ${this.token}` } : {};
    }

    async getTime() {
        // Get stored time for this device
        const response = await fetch(`${this.server}/time/${this.deviceId}`, {
            headers: this.getHeaders()
        });
        return response.json();
    }

    connect() {
        this.setupSocket();
        this.socket.connect();
    }
}

// Usage
const client = new KeepEduroamClient('provider', 'phone_001');
client.connect();

Error Handling

CodeMessageSolution
401 Unauthorized Valid JWT token required
403 Forbidden Cannot access another device's data
404 No providers available No providers online, wait
429 Rate limit exceeded Slow down requests
503 Service unavailable Server is down, retry later
Error handling example: 
socket.on('error', (data) => {
    console.error('❌ Error:', data.message);
    if (data.code === 'NO_SERVERS') {
        setTimeout(() => reconnect(), 5000);
    }
});

Security

🔑 Authentication

Device-based identification with JWT:

socket.emit('register_provider', {
    "id": "unique_device_id"
});

The server returns a JWT token that must be included in all HTTP requests.

🔒 Rate Limiting

100 requests per 60 seconds per IP

Exceeding the limit returns a 429 status.

🛡️ Device Isolation

Each device can only access its own data

403 Forbidden returned when accessing another device.

📡 HTTPS / TLS

All traffic is encrypted via Cloudflare

SSL/TLS enabled on all endpoints.
